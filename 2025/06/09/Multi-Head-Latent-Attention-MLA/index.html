Multi-Head Latent Attention(MLA)

发表于2025-06-09|更新于2025-06-10
|浏览量:

KV cache

在人工智能快速发展的今天，Transformer架构依然占据着主导地位。
在大模型时代，模型通过已生成的token来预测下一个token。
根据Transformer的架构特点，在自回归生成过程中需要重复使用之前注意力机制计算得到的Key-Value(KV)值。
然而，每次重新计算这些KV值会带来巨大的时间开销。
为了解决这个问题，我们可以将这些KV值缓存在显存中，以提高生成效率。这就kv缓存的提出的前提。
然而随着大模型的上下文长度需求越来越强，kv缓存的存储要求也越来越高，逐渐达到了单卡的显存上限，在这种情况下我们可以将单机的
多卡链接起来，更进一步将多机链接起来一起使用，但是这种方式计算效率是远不如单卡的，单卡间的通信带宽是同机多卡的3倍，是跨机的20倍。
这显然违背了kv缓存的初衷。因此针对这些问题，学术界提出了MQA和GQA.

MQA

在传统的多头注意力机制(Multi-Head Attention, MHA)中，每个注意力头都需要独立存储其对应的Key-Value缓存，这导致了显存使用量的线性增长。
为了解决这个问题，Google提出了多头查询注意力机制(Multi-Query Attention, MQA)，它允许多个查询头共享同一个Key-Value缓存，从而显著减少了显存占用。

GQA

但是MQA损失信息太多了，导致transformer的性能下降严重，为了尽可能避免性能的下降同时又缓解kv cache，我们需要在MHA和MQA中取平衡值。于是出现了
分组查询注意力 (Grouped-Query Attention, GQA),GQA将注意力头分为几个组，每个组共享KV值，这样也能降低kv cache的消耗。

MLA

然而，即使采用GQA方案，KV缓存仍然可能面临瓶颈。为了进一步优化，DeepSeek团队提出了多头部潜在注意力机制(Multi-Head Latent Attention, MLA)。
MLA 的核心思想是通过将输入序列投影到一个低维的潜在空间（Latent Space）来降低计算复杂度和显存占用。这个潜在空间的维度是一个关键的超参数。例如，在 DeepSeek 的相关论文中，其模型包含 56 个注意力头，而潜在空间的维度被设置为注意力头维度的4倍。通过这种低秩近似（Low-rank Approximation）处理，MLA 能够显著地减少显存占用。
在详细介绍MLA之前，让我们先回顾一下传统多头注意力机制的数学表达式：

对于输入序列 X∈Rn×dX \in \mathbb{R}^{n \times d}X∈Rn×d，其中 nnn 是序列长度，ddd 是隐藏维度，多头注意力机制的计算过程如下：

  1. 首先将输入 XXX 投影到查询(Q)、键(K)和值(V)空间：

    Q=XWQ,K=XWK,V=XWVQ = XW^Q, K = XW^K, V = XW^V Q=XWQ,K=XWK,V=XWV

  2. 将Q、K、V分别分割成 hhh 个头：

    Q=[Q1,Q2,...,Qh],K=[K1,K2,...,Kh],V=[V1,V2,...,Vh]Q = [Q_1, Q_2, ..., Q_h], K = [K_1, K_2, ..., K_h], V = [V_1, V_2, ..., V_h] Q=[Q1​,Q2​,...,Qh​],K=[K1​,K2​,...,Kh​],V=[V1​,V2​,...,Vh​]

  3. 对每个头计算注意力分数：

    Attention(Qi,Ki,Vi)=softmax(QiKiTdk)ViAttention(Q_i, K_i, V_i) = softmax(\frac{Q_iK_i^T}{\sqrt{d_k}})V_i Attention(Qi​,Ki​,Vi​)=softmax(dk​​Qi​KiT​​)Vi​

  4. 最后将所有头的输出拼接并投影：

    MultiHead(X)=Concat(Attention(Q1,K1,V1),...,Attention(Qh,Kh,Vh))WOMultiHead(X) = Concat(Attention(Q_1, K_1, V_1), ..., Attention(Q_h, K_h, V_h))W^O MultiHead(X)=Concat(Attention(Q1​,K1​,V1​),...,Attention(Qh​,Kh​,Vh​))WO

然而，将输入序列压缩到低维空间不可避免地会带来信息损失。一种潜在的优化思路是"矩阵吸收"（Matrix Absorption）。在标准的多头注意力中，注意力得分的核心计算为 QKT=(XWQ)(XWK)T=XWQ(WK)TXTQK^T = (XW^Q)(XW^K)^T = X W^Q (W^K)^T X^TQKT=(XWQ)(XWK)T=XWQ(WK)TXT。理论上，权重矩阵的乘积 WQ(WK)TW^Q (W^K)^TWQ(WK)T 可以被预先计算，从而简化后续计算。

但是，这个理想的优化过程忽略了位置编码的复杂性，特别是旋转位置编码（RoPE）。在应用RoPE时，Query 和 Key 的计算会包含一个与位置相关的旋转操作，其公式可示意为：(XiWQ⋅Ri)(XiWK⋅Ri)T(X_iW^Q \cdot R_i)(X_iW^K \cdot R_i)^T(Xi​WQ⋅Ri​)(Xi​WK⋅Ri​)T，其中 RiR_iRi​ 是位置 iii 的旋转矩阵。

关键在于，旋转操作 RiR_iRi​ 是依赖于 token 位置的，并且它作用于线性投影（XiWQX_iW^QXi​WQ 和 XiWKX_iW^KXi​WK）之后。这使得 RiR_iRi​ 无法与权重矩阵 WQW^QWQ 和 WKW^KWK 一起被提前"吸收"或合并，因此阻碍了矩阵吸收这一优化策略的实施。

为了解决这个冲突，DeepSeek 团队提出了解耦位置编码 (Decoupled Positional Encoding) 方案。其精髓在于将 Query (Q) 向量 和 Key-Value (KV) 压缩向量 都显式地拆分为两部分：

  1. 内容部分 (NoPE - No Position Embedding)：这部分向量承载主要的语义信息，不参与 RoPE 旋转。
  2. 位置部分 (RoPE - Rotary Position Embedding)：这部分向量专门用于编码位置信息，会参与 RoPE 旋转。

具体流程如下：

  • 对于 Query：输入的 hidden_states 经过映射后，会得到一个 Q 向量，这个 Q 向量被切分成 q_nope 和 q_rope 两部分。
  • 对于 Key & Value：输入的 hidden_states 被压缩成一个低维的 KV 压缩向量。这个压缩向量同样被切分：一部分用于生成 k_nope 和 v；另一部分则成为一个跨所有头共享的 k_rope 向量。
  • 旋转与计算：接下来，RoPE 的旋转操作只施加在 q_rope 和共享的 k_rope 上。最后，将旋转后的 q_rope 与 q_nope 拼接成最终的 Query，将旋转后的 k_rope 与 k_nope 拼接成最终的 Key。

通过这种方式，主要的语义向量（q_nope, k_nope）与位置旋转操作完全解耦，从而使得上游的矩阵可以被安全地"吸收"合并，实现了推理加速。
因此笔者认为Q的低秩压缩其实在理想情况是不需要的，但是由于位置编码的原因，需要将内容部分和内容部分对其，位置部分与位置部分对其，才需要对Q进行低秩压缩。综上所述，最终的MLA数学表达式为：

1. Query Projection (查询向量生成)

  • Down-projection: 首先，将输入隐状态 ht\mathbf{h}_tht​ 压缩成一个低维的查询潜在向量 ctQ\mathbf{c}_t^QctQ​。

    ctQ=WDQht\mathbf{c}_t^Q = W^{DQ}\mathbf{h}_t ctQ​=WDQht​

  • Up-projection & Splitting: 然后，使用这个潜在向量生成内容部分 qtC\mathbf{q}_t^CqtC​ (即 q_nope) 和位置部分 qtR\mathbf{q}_t^RqtR​ (即 q_rope)。位置部分会应用旋转位置编码。

    qtC=WUQctQ\mathbf{q}_t^C = W^{UQ}\mathbf{c}_t^Q qtC​=WUQctQ​

    qtR=RoPE(WQRctQ)\mathbf{q}_t^R = \text{RoPE}(W^{QR}\mathbf{c}_t^Q) qtR​=RoPE(WQRctQ​)

  • Final Query: 最终，每个注意力头 iii 的查询向量 qt,i\mathbf{q}_{t,i}qt,i​ 由其对应的内容和位置部分拼接而成。

    qt,i=[qt,iC;qt,iR]\mathbf{q}_{t,i} = [\mathbf{q}_{t,i}^C; \mathbf{q}_{t,i}^R] qt,i​=[qt,iC​;qt,iR​]

2. Key & Value Projection (键值向量生成)

  • Down-projection: 同样地，将输入隐状态 ht\mathbf{h}_tht​ 压缩成一个共享的低维KV潜在向量 ctKV\mathbf{c}_t^{KV}ctKV​。

    ctKV=WDKVht\mathbf{c}_t^{KV} = W^{DKV}\mathbf{h}_t ctKV​=WDKVht​

  • Up-projection for Key (Content Part): 从KV潜在向量中生成Key的内容部分 ktC\mathbf{k}_t^CktC​ (即 k_nope)。

    ktC=WUKctKV\mathbf{k}_t^C = W^{UK}\mathbf{c}_t^{KV} ktC​=WUKctKV​

  • RoPE Part of Key: Key的位置部分 ktR\mathbf{k}_t^RktR​ (即 k_rope) 则是直接从原始隐状态 ht\mathbf{h}_tht​ 生成并应用旋转编码，这部分在所有头之间共享。

    ktR=RoPE(WKRht)\mathbf{k}_t^R = \text{RoPE}(W^{KR}\mathbf{h}_t) ktR​=RoPE(WKRht​)

  • Final Key: 每个头 iii 的最终Key向量由其内容部分和共享的位置部分拼接而成。

    kt,i=[kt,iC;ktR]\mathbf{k}_{t,i} = [\mathbf{k}_{t,i}^C; \mathbf{k}_t^R] kt,i​=[kt,iC​;ktR​]

  • Value Projection: Value向量 vtC\mathbf{v}_t^CvtC​ 也从共享的KV潜在向量生成。

    vtC=WUVctKV\mathbf{v}_t^C = W^{UV}\mathbf{c}_t^{KV} vtC​=WUVctKV​

3. Attention Calculation (注意力计算)

  • Attention Score: 计算每个头 iii 的注意力输出 ot,i\mathbf{o}_{t,i}ot,i​。

    ot,i=∑j=1tSoftmaxj(qt,iTkj,idhC+dhR)vj,iC\mathbf{o}_{t,i} = \sum_{j=1}^{t} \text{Softmax}_j(\frac{\mathbf{q}_{t,i}^T \mathbf{k}_{j,i}}{\sqrt{d_h^C + d_h^R}})\mathbf{v}_{j,i}^C ot,i​=j=1∑t​Softmaxj​(dhC​+dhR​​qt,iT​kj,i​​)vj,iC​

  • Final Output: 将所有头的输出拼接后，通过一个最终的投影矩阵得到模块的输出 ut\mathbf{u}_tut​。

    ut=WO[ot,1;ot,2;… ;ot,nh]\mathbf{u}_t = W^O[\mathbf{o}_{t,1}; \mathbf{o}_{t,2}; \dots; \mathbf{o}_{t,n_h}] ut​=WO[ot,1​;ot,2​;…;ot,nh​​]

符号含义

  • ht\mathbf{h}_tht​: 第 ttt 个 token 的输入隐状态 (input hidden state)。
  • ctQ\mathbf{c}_t^QctQ​: 查询 (Query) 的低维潜在向量。
  • ctKV\mathbf{c}_t^{KV}ctKV​: 键 (Key) 和值 (Value) 共享的低维潜在向量。
  • WDQ,WDKVW^{DQ}, W^{DKV}WDQ,WDKV: 分别是Q和KV的下投影（压缩）矩阵。
  • WUQ,WUK,WUVW^{UQ}, W^{UK}, W^{UV}WUQ,WUK,WUV: 分别是Q、K、V的上投影（解压）矩阵。
  • WQR,WKRW^{QR}, W^{KR}WQR,WKR: 分别是用于生成Q和K的位置部分（rope part）的投影矩阵。
  • qtC,ktC,vtC\mathbf{q}_t^C, \mathbf{k}_t^C, \mathbf{v}_t^CqtC​,ktC​,vtC​: 分别对应Q、K、V的内容部分 (content part)，即您提到的 q_nope, k_nope。
  • qtR,ktR\mathbf{q}_t^R, \mathbf{k}_t^RqtR​,ktR​: 分别对应Q、K的位置部分 (RoPE part)，即您提到的 q_rope, k_rope。
  • RoPE(⋅)\text{RoPE}(\cdot)RoPE(⋅): 旋转位置编码函数。
  • qt,i,kj,i,vj,iC\mathbf{q}_{t,i}, \mathbf{k}_{j,i}, \mathbf{v}_{j,i}^Cqt,i​,kj,i​,vj,iC​: 分别是第 iii 个注意力头在时间步 ttt 或 jjj 的最终Q, K, V向量。
  • dhC,dhRd_h^C, d_h^RdhC​,dhR​: 分别是每个注意力头中，内容部分和位置部分的维度。
  • ot,i\mathbf{o}_{t,i}ot,i​: 第 iii 个注意力头的输出。
  • WOW^OWO: 最终的输出投影矩阵。
  • ut\mathbf{u}_tut​: 第 ttt 个 token 的最终输出。
