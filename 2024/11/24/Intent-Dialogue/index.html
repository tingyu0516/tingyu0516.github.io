面向多轮意图分类的意图感知对话生成与多任务对比学习

发表于2024-11-24|更新于2024-11-25
|浏览量:

论文地址：https://arxiv.org/pdf/2411.14252

动机

数据集的构造困难

在聊天机器人系统中，生成大规模、特定领域、多语言多轮对话数据集仍然是训练有效多轮意图分类模型的一个重大障碍。训练有效的多轮意图分类（Multi-Turn Intent Classification, MTIC）模型需要大量的领域特定、多语言的多轮对话数据集。然而，生成这样的数据集是一个巨大的挑战，因为手动标注多轮对话中的意图非常耗时且资源密集。
特别是在多语言环境中，数据的收集和标注更加困难，导致数据稀缺。

多轮对话任务的准确度不足

基于传统的单轮数据的多轮训练方法无法充分利用上下文信息，会导致模型在多轮对话中识别用户意图的准确性较低。

贡献

意图链（Chain-of-Intent）

提出了一种新的方法，通过结合隐马尔可夫模型（HMM）和大型语言模型（LLM）来自动生成情境感知的、意图驱动的多轮对话。这种方法可以从历史聊天记录中提取领域特定的知识，生成真实的意图序列，并利用LLM生成高质量的多语言对话。
通过自动生成大规模、领域特定的对话数据集，减少了对大量手动标注数据的依赖。

MINT-CL

提出了一个用于多轮意图分类的多任务对比学习框架。该框架通过利用当前和上下文信息，并采用多任务对比学习目标来优化模型，提高了多轮意图分类的准确性。

MINT-E

发布了一个多语言、意图感知的多轮电子商务对话语料库MINT-E，支持该领域的未来研究。该语料库涵盖了八个不同市场的多样语言和大量意图，为开发和评估多轮意图分类模型提供了宝贵资源。

理论方法

Chain-of-Intent (意图链生成)

Chain-of-Intent 是一种结合隐马尔可夫模型（HMM）和大型语言模型（LLM）生成意图驱动对话的方法。通过从历史聊天记录中提取领域特定知识，生成真实的意图序列，并利用 LLM 生成高质量的多轮对话。

领域知识提取：从实际聊天记录中提取每段对话的轮数分布 (P(T))( P(T) )(P(T)) 和意图转换矩阵 (P(It∣It−1))( P(I_t | I_{t-1}) )(P(It​∣It−1​))。

意图采样：使用 HMM 模型，根据 (P(T))( P(T) )(P(T)) 采样对话轮数 ( T )，根据 (P(I1))( P(I_1) )(P(I1​)) 采样初始意图 (I1)( I_1 )(I1​)，并根据 (P(It∣It−1))( P(I_t | I_{t-1}) )(P(It​∣It−1​)) 采样后续意图 (It)( I_t )(It​)。

对话生成：利用 LLM 生成与历史对话上下文一致的问题和答案。每个问题 (qt)( q_t )(qt​) 的生成基于当前意图 (It)( I_t )(It​) 和历史对话 (ht)( h_t )(ht​)。

MINT-CL (多任务对比学习框架)

方法介绍：MINT-CL 是一个用于多轮意图分类的多任务对比学习框架。通过结合当前和历史信息，并使用多任务对比学习目标优化模型，提高意图分类的准确性。
关键步骤：
响应排名任务：为每段对话的最后一个轮次生成两个响应：高质量响应 ( a^+ ) 和低质量响应 ( a^- )。这些响应通过不同的模型或配置生成。
对比损失：使用对比损失函数 ( L_{\text{contrastive}} ) 鼓励模型为高质量响应分配更高的分数。具体公式如下：[Lcontrastive=−log⁡exp⁡(L2+)exp⁡(L2+)+exp⁡(L2−)][ L_{\text{contrastive}} = -\log \frac{\exp(L_2^+)}{\exp(L_2^+) + \exp(L_2^-)} ][Lcontrastive​=−logexp(L2+​)+exp(L2−​)exp(L2+​)​]其中， [L2+=H2+W2+b2和L2−=H2−W2+b2](H2+)和(H2−)[ L_2^+ = H_2^+ W_2 + b_2 \quad \text{和} \quad L_2^- = H_2^- W_2 + b_2 ] ( H_2^+ ) 和 ( H_2^- )[L2+​=H2+​W2​+b2​和L2−​=H2−​W2​+b2​](H2+​)和(H2−​)分别是高质量和低质量响应的编码表示。
联合训练目标：最终的损失函数是意图分类损失 (Lintent)( L_{\text{intent}} )(Lintent​)和对比损失 (Lcontrastive)( L_{\text{contrastive}} )(Lcontrastive​) 的加权和： [L=Lintent+λLcontrastive][ L = L_{\text{intent}} + \lambda L_{\text{contrastive}} ][L=Lintent​+λLcontrastive​] 其中，(λ)( \lambda )(λ) 是平衡对比损失贡献的超参数，通过手动调参确定最佳值为 0.3。
