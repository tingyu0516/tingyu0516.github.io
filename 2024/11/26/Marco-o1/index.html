Marco-o1：迈向用于开放式解决方案的开放推理模型

发表于2024-11-26|更新于2024-11-26
|浏览量:

论文地址：https://arxiv.org/pdf/2411.14405

动机

该论文旨在探讨大型推理模型（LRM）在开放领域中的应用能力，特别是如何有效地推广到那些没有明确标准答案且奖励难以量化的更广泛领域。即模型的推理能力不能仅局限了数学、物理和编程等领域，还要重视开放领域的解决方案。

方法

具体来说，论文通过构建和优化Marco-o1模型，探索了在开放性问题解决中的应用。研究团队使用了链式思维（Chain-of-Thought, CoT）微调、蒙特卡洛树搜索（Monte Carlo Tree Search, MCTS）、反思机制以及创新的推理策略，以增强模型处理复杂现实任务的能力。这些技术的应用旨在测试模型是否能够在没有明确标准答案的情况下，通过扩展解决方案空间来找到正确的答案，并提高其在不同任务中的表现，如数学、物理、编程以及自然语言处理等领域的复杂问题解决能力。

理论实施

Chain-of-Thought (CoT) 微调

使用开放源代码的 CoT 数据集和自开发的合成数据对基础模型进行全参数微调，以增强模型的推理能力。
通过监督微调策略使用多种数据集，包括过滤后的 Open-O1 CoT 数据集、合成的 Marco-o1 CoT 数据集和 Marco Instruction 数据集，来提升模型的推理能力。
Monte Carlo Tree Search (MCTS) 集成：

将大型语言模型与 MCTS 结合，利用模型的输出置信度引导搜索，扩展解空间。
在 MCTS 框架中，每个节点代表问题解决过程中的一个推理状态，可能的动作是从节点生成的 LLM 输出，这些输出代表潜在的推理步骤或小步骤。
通过 rollout 阶段继续推理过程到终止状态，并计算奖励分数 ( R )，用于评估和选择有希望的路径。
计算每个 token 的置信度分数 (ci)( c_i )(ci​):[ci=exp⁡(p(ti))∑k=15exp⁡(p(tk))][ c_i = \frac{\exp(p(t_i))}{\sum_{k=1}^{5} \exp(p(t_k))} ][ci​=∑k=15​exp(p(tk​))exp(p(ti​))​] 其中 (p(ti))( p(t_i) )(p(ti​)) 是第 ( i ) 个 token 的对数概率，(p(tk))( p(t_k) )(p(tk​)) 是第 ( i ) 步预测的前 5 个 token 的对数概率。

推理动作策略

探索不同粒度的 MCTS 搜索，包括使用完整推理步骤作为动作（step）和使用 32 或 64 个 token 的小步骤作为动作（mini-step）。
引入反思机制，在每个思考过程的末尾添加提示语句：“Wait! Maybe I made some mistakes! I need to rethink from scratch.”，促使模型自我反思和重新评估其推理步骤。

实验验证

基于 Qwen2-7B-Instruct 进行监督微调，创建 Marco-o1-CoT。
使用不同的动作粒度（step 和 mini-step）在 MCTS 框架下测试模型，包括 Marco-o1-MCTS (step)、Marco-o1-MCTS (mini-step of 64 tokens) 和 Marco-o1-MCTS (mini-step of 32 tokens)。
在 MGSM 数据集的英文和中文子集上测试这些配置，结果显示 MCTS 增强的模型在推理任务中表现出显著改进。

翻译任务应用

在翻译任务中展示了 Marco-o1 模型的能力，特别是在处理俚语和口语表达时，模型能够提供更准确和自然的翻译。

实验设计

数据集与训练策略

数据集

Open-O1 CoT 数据集（过滤版）：从 Open-O1 项目的数据集中筛选出高质量的链式思维（CoT）数据，以提高模型的结构化推理能力。

Marco-o1 CoT 数据集（合成版）：使用蒙特卡洛树搜索（MCTS）生成的复杂推理路径数据，进一步提升模型的推理能力。
Marco 指令数据集：包含一系列指令跟随数据，确保模型在执行复杂任务时保持高效和通用性。

训练策略

监督细调（SFT）：使用上述数据集对基础模型进行全参数细调，以增强其推理能力。

解决方案空间扩展

蒙特卡洛树搜索（MCTS）

节点作为推理状态：在 MCTS 框架中，每个节点代表问题解决过程中的一个推理状态。
动作作为 LLM 输出：从节点出发的可能动作由 LLM 生成，这些输出代表推理链中的潜在步骤或小步骤。
rollout 和奖励计算：在 rollout 阶段，LLM 继续推理过程直到终端状态，并计算每个状态的价值。
引导 MCTS：使用奖励分数 ( R ) 评估和选择有希望的路径，引导搜索方向。

信心评分计算

对于每个生成的 token (ti)( t_i )(ti​)，计算其信心评分 (ci)( c_i )(ci​)： [ci=exp⁡(p(ti))∑k=15exp⁡(p(tk))][ c_i = \frac{\exp(p(t_i))}{\sum_{k=1}^5 \exp(p(t_k))} ][ci​=∑k=15​exp(p(tk​))exp(p(ti​))​] 其中 (p(ti))( p(t_i) )(p(ti​)) 是第 ( i ) 个 token 的对数概率，(p(tk))( p(t_k) )(p(tk​)) 是前 5 个预测 token 在第 ( i ) 步的对数概率。
计算所有 token 的平均信心评分 (v)( v )(v)： [v=1n∑i=1nci][ v = \frac{1}{n} \sum_{i=1}^n c_i ][v=n1​∑i=1n​ci​] 这个平均值作为奖励信号，评估推理路径的质量。

推理行动策略

动作选择

步骤作为动作：允许模型生成完整的推理步骤作为动作，每个 MCTS 节点代表一个完整的思考或动作标签。
小步骤作为动作：使用 32 或 64 个 token 的小步骤作为动作，增加解空间的探索范围，提高模型处理复杂推理任务的能力。

反思机制

在每次思考过程结束时，添加提示语句“等等！我可能犯了一些错误！我需要重新思考。”，促使模型自我反思和重新评估推理步骤。这显著提高了模型在解决难题时的准确性。

实验结果

实验设置

基于 Qwen2-7B-Instruct 模型，使用训练数据进行 SFT，创建 Marco-o1-CoT 模型。
在 MCTS 树搜索框架中应用不同的动作粒度：
Marco-o1-MCTS (step)：使用每个推理步骤作为动作。

Marco-o1-MCTS (mini-step of 64 tokens)：使用 64 个 token 的小步骤作为动作。

Marco-o1-MCTS (mini-step of 32 tokens)：使用 32 个 token 的小步骤作为动作。

实验结果

在 MGSM 数据集的英文和中文子集上测试了不同配置的模型，结果如下：
MGSM-En (Acc.)：

Qwen2-7B-Instruct: 84.23%

Marco-o1-CoT: 85.60%

Marco-o1-MCTS (step): 90.40%

Marco-o1-MCTS (mini-step of 64 tokens): 88.40%

Marco-o1-MCTS (mini-step of 32 tokens): 87.60%

MGSM-Zh (Acc.)：

Qwen2-7B-Instruct: 76.80%

Marco-o1-CoT: 71.20%

Marco-o1-MCTS (step): 80.00%

Marco-o1-MCTS (mini-step of 64 tokens): 80.40%

Marco-o1-MCTS (mini-step of 32 tokens): 82.40%
